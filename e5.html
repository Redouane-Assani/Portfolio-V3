Administration des Systèmes et Réseaux (Épreuve E5)

L'épreuve E5 certifie les compétences en conception, déploiement et sécurisation d'infrastructures systèmes et réseaux. Elle s'appuie sur deux projets réalisés en maquette sur Proxmox dans le cadre du BTS SIO SISR.

Projet 1 — Infrastructure réseau sécurisée RÉALISÉ

Contexte

Mise en place d'une infrastructure réseau complète pour une PME fictive, intégrant une solution de stockage NAS, un pare-feu, un hyperviseur et une supervision centralisée des journaux.

Équipements déployés

Firewall Stormshield

  • Routage inter-VLAN (firewall-on-a-stick)
  • Politique de filtrage par sections
  • NAT / accès Internet
  • DHCP pour le VLAN 2090

Switch Cisco CBS220

  • VLAN 2090 — LAN principal
  • VLAN 2092 — DMZ
  • Lien trunk taggé vers le Stormshield

Serveur Proxmox

  • Hyperviseur de la maquette
  • Hébergement des VMs (client, log)
  • Bridges réseau avec tag VLAN

TrueNAS

  • NAS réseau (partages SMB/NFS)
  • Stockage centralisé
  • Envoi des logs vers le serveur Syslog

VM Rsyslog (supervision)

  • Centralisation des journaux
  • Sources : Stormshield + TrueNAS
  • VLAN 2090 — 172.16.209.x

VM Cliente

  • Poste client de test (VLAN 2090)
  • Adresse DHCP via le Stormshield

Projet 2 — Active Directory & Wi-Fi 802.1X RÉALISÉ

Contexte

Une PME de 40 employés souhaite centraliser l'authentification de ses utilisateurs et sécuriser l'accès à son réseau sans-fil. Le projet s'appuie sur l'infrastructure du projet 1 (pare-feu, switch, Proxmox, Rsyslog) et y ajoute un contrôleur de domaine Windows Server et des bornes Wi-Fi Cisco configurées en 802.1X.

Architecture technique

Vert = Nouveau dans ce projet  |  Orange = Réutilisé du projet 1

VM SRV-AD-01 — Windows Server 2025

  • Active Directory DS + DNS + DHCP
  • NPS (serveur RADIUS intégré)
  • Domaine : simcorp.local
  • IP fixe : 172.16.213.10 (VLAN 2093)

Bornes Wi-Fi Cisco

  • Mode 802.1X — EAP-PEAP
  • Rôle d'authenticator
  • Relaient les requêtes vers le NPS
  • IP de gestion : 172.16.211.10 (VLAN 2091)

Stormshield — Nouvelles règles

  • Interfaces VLAN 2091 et 2093 ajoutées
  • DHCP pour VLAN 2091 et 2093
  • Filtrage inter-VLAN (isolation Wi-Fi)

Switch Cisco CBS220 — Mis à jour

  • VLAN 2091 (Wi-Fi) ajouté
  • VLAN 2093 (Serveurs) ajouté
  • Trunk mis à jour vers Stormshield et Proxmox

VM Rsyslog — Étendue

  • Sources ajoutées : AD + NPS
  • Centralise les authentifications Wi-Fi
  • Règle FW : UDP 514 de VLAN 2093 → 2090

TrueNAS — Étendu

  • Stockage des profils itinérants AD
  • Partage réseau accessible depuis le domaine

Fonctionnement du 802.1X — Flux d'authentification

Lorsqu'un utilisateur se connecte au Wi-Fi, le flux est le suivant :

① Le client Wi-Fi demande l'accès au réseau sans-fil.
② La borne Cisco bloque le trafic et demande des identifiants (EAP-PEAP).
③ La borne relaye les identifiants vers le serveur NPS (RADIUS) — UDP 1812.
④ Le NPS interroge l'Active Directory : le compte existe-t-il ? Est-il dans le groupe GRP_WiFi-Autorises ?
⑤ OUI → accès accordé, le client obtient une IP dans le VLAN 2091.
⑤ NON → connexion rejetée, aucun accès réseau.

Chaque connexion est nominative, journalisée dans le Rsyslog et révocable en désactivant simplement le compte AD.

Politique de filtrage Stormshield — VLAN 2091 & 2093

Les règles sont lues de haut en bas. Le premier match s'applique.

PASS  | VLAN-WiFi → SRV-AD-01   | UDP 1812  | Authentification RADIUS        # EN PREMIER
BLOCK  | VLAN-WiFi → VLAN-Serveurs | Any      | Wi-Fi interdit aux serveurs
BLOCK  | VLAN-WiFi → LAN-Clients   | Any      | Wi-Fi interdit au LAN
BLOCK  | VLAN-Serveurs → VLAN-WiFi | Any      | Serveurs interdits vers Wi-Fi
PASS  | VLAN-Serveurs → Rsyslog   | UDP 514  | Logs AD/NPS → Syslog
PASS  | LAN-Clients → VLAN-Serveurs| Any      | LAN accède à l'AD
PASS  | VLAN-WiFi → Internet     | Any      | Navigation Web clients Wi-Fi
PASS  | VLAN-Serveurs → Internet  | Any      | Serveurs accès Internet
BLOCK  | VLAN-WiFi → Any         | Any      | Isolation résiduelle Wi-Fi
